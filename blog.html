• Scope of var, let, and const
  • Var :

    • Function Scope: When we declare a variable with var, it can only be used inside the function where it's declared.

    Hoisting: JavaScript moves the var variable declarations to the top of their containing function or the global context.

    Redeclaration Allowed: We can declare the same var variable name multiple times within the same function or global scope without any errors. The later declarations effectively override the earlier ones.

  • let :

    • Block Scope: It can only be used inside the block where it's declared. It won't be accessible outside of that block.

    Mutable Value:let variables allow us to change (reassign) their values after they have been initially declared and assigned.

    No Redeclaration: We can't declare the same let variable name multiple times within the same block or scope.

  • let :

    • Block Scope: It can only be used inside the block where it's declared. It won't be accessible outside of that block.

    Immutable Value:If we assigned a value once,it cannot be reassigned to a new value. They are considered constant or immutable.

    No Redeclaration: We can't declare the same let variable name multiple times within the same block or scope.

  • Use cases of null and undefined
  • null :

    • Intentional absence of Value:null is a value that represents the absence of an object or value.It is often used to indicate that a variable, object property, or function return value intentionally has no value or is empty.

    Clearing Value:We can set a variable to null to clear or reset its value, especially in objects or data structures.

  • Undefined :

    • Missing Object Properties:If we want to access a property of an object that does not exist, it returns undefined. This is useful for checking the existence of properties in objects.

    Array Elements that Don't Exist:When we access an element in an array at an index that doesn't exist, it returns undefined.

    Accessing Object Properties That Are Not Present:When you want to access a property of an object that doesn't exist, it also returns undefined.

  • REST API

    • REST API is a way of accessing web services in a simple and flexible way without having any processing.REST API is an API that conforms to the design principles of the REST, or representational state transfer architectural style.REST APIs are designed to allow different software applications to communicate with each other over the internet by making requests and receiving responses using standard HTTP methods and data formats.

    Resources:In REST, everything is treated as a resource. Resources can be data objects, services, or even entities like users, products, or orders.

    Representation:Data exchanged between the client and server is typically in a standardized format, such as JSON or XML.

    HTTP Methods:REST APIs use standard HTTP methods to perform actions on resources. The most commonly used HTTP methods in REST are:GET,POST,PUT,DELETE.

    Uniform Interface:REST APIs have a uniform and consistent interface, which simplifies interactions. Clients interact with resources using standard HTTP methods and URLs.

    HTTP Status Codes:REST APIs use HTTP status codes to indicate the result of a request. Common status codes are:200 OK, 201 Created,204 No Content,400 Bad Request,401 Unauthorized,403 Forbidden,404 Not Found,500 Internal Server Error.
